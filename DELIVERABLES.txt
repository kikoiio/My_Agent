ARCHITECTURE HIGHLIGHTS
=======================

1. MEMORY SYSTEM
   - L1 (Session): Fast transient, 12K tokens per persona
   - L2 (Episodic): SQLite FTS5, searchable
   - L3 (Dreams): Consolidated patterns, quality-gated
   - Auto-consolidation: idle/pending/cron triggers

2. EVALUATION FRAMEWORK
   - 5-LLM jury with independent scoring
   - Debate phase for disagreement resolution
   - Gold-standard blind probes for calibration
   - Multiplicative weight updates (×1.01/×0.97)

3. SECURITY
   - Injection detection with risk scoring
   - External content wrapping with trust levels
   - Atomic SQL rate limiting (no race conditions)
   - Biometric gates: Face (InsightFace) + Voice (3D-Speaker)

4. ORCHESTRATION
   - LangGraph draft→critic→respond pipeline
   - Rule-based message routing
   - Circuit breaker with loop detection
   - Distributed tracing (SQLite)

5. HARDWARE ABSTRACTION
   - Platform-agnostic HAL interface
   - RPi implementation (picamera2, sherpa-onnx, openwakeword)
   - Remote proxy via WebSocket (headless operation)
   - Fallback chain: local → remote → null

6. DEPLOYMENT
   - Docker Compose for backend services
   - systemd unit for edge runtime
   - WireGuard VPN for secure Pi↔Backend tunnel
   - Hardware verification script
